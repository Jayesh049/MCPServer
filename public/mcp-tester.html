Free API Tokens You Need
🤗
Hugging Face — Pneumonia imaging
Enables lxyuan/vit-xray-pneumonia-classification — a real ViT image classifier. Free tier: 30,000 requests/month.
HF_API_TOKEN=hf_xxxxxxxxxxxxxxxx
→ huggingface.co/settings/tokens (free)
⚡
Groq — Patient Chat + Healer (Free)
Llama 3.1 70B via Groq. Fastest free LLM — ~6,000 tokens/min. Used by patient chat and auto-healer. Recommended first choice.
GROQ_API_KEY=gsk_xxxxxxxxxxxx
→ console.groq.com/keys (free)
🌊
Google Gemini — Chat + Healer (Free)
Gemini 1.5 Flash. 1 million tokens/day free. Used if GROQ key is not set. Reliable fallback for patient chat.
GEMINI_API_KEY=AIzaxxxxxxxx
→ aistudio.google.com/apikey (free)
🗄
Neon — PostgreSQL (Free)
Free Postgres for Q&A history, RAG embeddings, healer fix-pattern cache, and ML model artifacts. 0.5 GB free tier.
DATABASE_URL=postgresql://neon…
→ neon.tech (free)
🔧
Flask ML Sidecar — Imaging (Local)
Run the Flask sidecar locally to enable real sklearn predictions for all 11 imaging diseases. Ingest PDFs, then train per slug.
DISEASE_ML_URL=http://localhost:5001
DISEASE_ML_SECRET=your-secret-key
cd ml && pip install -r requirements.txt && python wsgi.py
🤖
Anthropic Claude — Optional (Paid)
Only used if GROQ and GEMINI keys are not set. Claude Haiku is cheapest at ~$0.80/M tokens. Best code-fix quality for healer.
ANTHROPIC_API_KEY=sk-ant-…
→ console.anthropic.com (paid)
All Data Sources
📖
Wikipedia RAG
Patient chat fetches Wikipedia medical articles, chunks + embeds via Transformers.js (free), stores in Postgres, retrieves by cosine similarity per question.
Refreshed per question
🧬
Disease Registry
20 diseases in src/diseases/registry.ts. Clinical diseases use validated scoring formulas. Imaging uses Flask sidecar or brightness/contrast stub.
Local · src/diseases/
📄
PDF Extraction
PDFs parsed locally via pdf-parse. Text matched to weighted keyword dictionaries (detectDiseasesFromText.ts). No PHI leaves the server.
Local · keyword weighted
👨‍⚕️
Synthetic Care Plans
All doctors, hospitals, and medications are fictional. Generated with a date-seeded RNG — rotates daily from src/care/dataPools.ts.
Synthetic · daily refresh
🔧
Auto-Healer (3 processes)
Watches Prisma, backend, and frontend simultaneously. Runtime errors → LLM generates patch → applies to source file → cached in Postgres.
LLM-powered · all processes
🌐
MCP / FHIR Tools
When MCP transport is active the app exposes FHIR care-gap analysis tools via a synthetic adapter. No real EHR data.
