Where does every piece of data come from?
Every data source is labeled throughout this interface — you always know exactly what you're seeing and why. This platform is for medical education only.
Data Sources
📖
Wikipedia RAG
Patient chat fetches Wikipedia medical articles, chunks + embeds via Transformers.js (free), stores in Postgres, retrieves by cosine similarity.
Refreshed per question
🧬
Disease Registry
20 diseases in registry.ts. Clinical diseases use Framingham, CKD-EPI, NAFLD score, CHA₂DS₂-VASc — real validated scoring.
Local · real models
📄
PDF Extraction
PDFs parsed locally via pdf-parse. Text matched to weighted keyword dictionaries. No PHI leaves the server.
Local · keyword weighted
👨‍⚕️
Synthetic Care Plans
All doctors, hospitals, and medications are fictional. Generated with a date-seeded RNG — plan data rotates daily from dataPools.ts.
Synthetic · daily refresh
🗄
Postgres / Neon
Q&A history, RAG embeddings, and healer fix-pattern cache all live in Postgres. Neon free tier for production (Render).
Postgres · Neon
🤖
LLM Providers
Chat uses whichever free or paid key you set: Anthropic Claude, Gemini Flash (free), Groq/Llama 70B (free), or OpenAI. Active provider shown in chat.
Anthropic · Gemini · Groq
🔧
Auto-Healer
Runtime errors from all 3 processes (Prisma, backend, frontend) are parsed, sent to an LLM, patched, and cached in Postgres automatically.
LLM-powered
🌐
MCP / FHIR Tools
When MCP transport is active the app exposes FHIR care-gap analysis tools via a synthetic adapter. No real EHR connection.
MCP · Synthetic FHIR
